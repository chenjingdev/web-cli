API Reference

Internal API documentation for the project management platform.

Authentication

All API requests require a Bearer token in the Authorization header.

Authorization: Bearer <your-api-token>

Tasks

GET /api/v1/tasks

List all tasks with optional filtering.

Query Parameters

ParameterTypeDescription
statusstringFilter by status: todo, in-progress, review, done
assigneestringFilter by assignee ID
pagenumberPage number (default: 1)
limitnumberItems per page (default: 20)
POST /api/v1/tasks

Create a new task.

Request Body

{
  "title": "Implement login page",
  "description": "Build the login page with OAuth support",
  "status": "todo",
  "priority": "high",
  "assigneeId": "usr_abc123"
}
PUT /api/v1/tasks/:id

Update an existing task.

DELETE /api/v1/tasks/:id

Delete a task by ID.

Members

GET /api/v1/members

List all project members.

POST /api/v1/members

Add a new member to the project.
