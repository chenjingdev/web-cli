# 캔버스 지원 전략

작성일: 2026-03-28

## 문제

agrune은 DOM 어노테이션 기반(`data-agrune-*`)으로 타깃을 감지한다. 캔버스 기반 UI에서는 이 방식이 동작하지 않거나 제한적이다.

| 유형 | 예시 | DOM 존재 | 현재 agrune |
|------|------|----------|------------|
| SVG 기반 | React Flow 엣지, D3 차트 | O (접근 어려움) | 스냅샷 누락 |
| HTML5 Canvas | 그림판, 화이트보드 | X (픽셀만) | 인식 불가 |
| WebGL | 3D 뷰어, 지도 | X | 인식 불가 |

`agrune_pointer`로 좌표 기반 조작은 가능하지만, AI가 캔버스 위에 무엇이 있는지 알 수 없는 것이 핵심 문제.

## 풀어야 할 질문

1. AI에게 캔버스 내용을 어떻게 전달할 것인가?
   - 스크린샷 기반 시각 인식
   - 앱이 제공하는 구조화된 메타데이터
   - 접근성 트리 활용
   - 하이브리드
2. 캔버스 유형별 현실적 지원 범위
   - SVG: DOM 자동 감지 가능성 높음
   - Canvas/WebGL: 앱 협조 또는 스크린샷 필수
3. 앱 개발자가 제공해야 하는 최소 인터페이스는?
4. 기존 `data-agrune-group` + `data-agrune-canvas`와의 관계
